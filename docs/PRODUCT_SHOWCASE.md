# 产品界面展示

本页用于在 GitHub 直接展示 AI Hot Spot 的核心页面效果。

## 项目展示概览

这组截图已经覆盖了项目的核心使用链路：

1. 热点总览与搜索
2. 关键词管理与新增
3. 采集策略配置
4. 邮件推送与规则设置

## 展示顺序

### 1. 热点总览

![热点总览](./screenshots/hotspot_mainpage.jpg)

该页面作为项目主界面，直接展示系统的核心能力：

- 顶部导航
- 热点卡片列表
- 排序与筛选控件
- 热度/相关度/可信度标签

这一页体现了系统如何把抓取到的内容转化为结构化热点列表，并支持按来源、时间、状态、重要性进行筛选。

### 2. 搜索与检索体验

![搜索功能](./screenshots/search_function.jpg)

这一页展示“监控 + 搜索”双模式能力：

- 支持直接输入关键词进行检索
- 支持全网搜索与本地搜索切换
- 搜索结果仍然保留三维评分与卡片化展示
- 同时支持被动监控与主动探索

### 3. 关键词管理

![关键词管理](./screenshots/keywords.jpg)

该页面展示关键词配置能力：

- 关键词列表
- 激活/暂停状态
- 新增关键词交互区

这是系统的入口配置页面，用于定义监控主题。

### 4. 新增关键词后的状态

![新增关键词后的管理页](./screenshots/keywords_afteradd.jpg)

这张图展示新增关键词后的状态变化：

- 新增关键词后总数从 2 变为 3
- 监控中的关键词数量同步变化
- 新增项直接进入列表并可被系统定时扫描

这一页说明系统不是静态原型，而是可操作、可验证的可运行产品。

### 5. 采集策略设置

![采集策略设置](./screenshots/settings.jpg)

该页展示后台策略控制能力：

- 严格 / 平衡 模式切换
- 内容时效窗口
- 免费信息源启用状态
- Twitter/X 高门槛过滤条件

这一页体现了对数据质量和信噪比控制的产品设计。

### 6. 邮件推送设置

![规则与通知设置](./screenshots/settings2.jpg)

这个页面补齐了策略设置的下半部分：

- 域名拒绝名单与优先名单
- 可选付费接口开关
- 保存全站配置、发送测试邮件、恢复默认配置

这一页体现了系统在规则配置、通知测试和运营控制上的完整性。
